01

Introducción

Objetivo del manual

Este manual describe de forma detallada y ordenada el proceso completo para instalar, configurar y ejecutar la aplicación Presupuesto App en un entorno de desarrollo local. El objetivo es permitir que cualquier desarrollador o técnico con conocimientos básicos en Flutter pueda poner en funcionamiento el sistema en su máquina.

Alcance

  • Instalación de todas las herramientas necesarias (Flutter SDK, Dart, Android Studio o VS Code).
  • Clonación y configuración del proyecto.
  • Instalación de dependencias.
  • Generación de código automático con build_runner.
  • Ejecución del sistema en un emulador o dispositivo físico.
  • Verificación de que la instalación funciona correctamente.
  • Solución a los problemas más comunes durante la instalación.
Fuera del alcance: Este manual no cubre el despliegue en tiendas de aplicaciones (Google Play, App Store) ni la configuración de servidores.

Público objetivo

  • Desarrolladores Flutter con nivel básico o intermedio.
  • Técnicos responsables de instalar o mantener el sistema.
  • Estudiantes que necesiten ejecutar el proyecto en su máquina local.

03

Instalación de dependencias

3.1 Instalar Flutter SDK

  1. Acceder a la página oficial de Flutter: https://docs.flutter.dev/get-started/install
  2. Seleccionar el sistema operativo correspondiente (Windows, macOS o Linux).
  3. Descargar el archivo comprimido del SDK.
  4. Extraer el contenido en una ruta sin espacios ni caracteres especiales.

Ejemplo de ruta en Windows:

C:\desarrollo\flutter

Agregar flutter\bin a la variable de entorno PATH. En Windows (PowerShell como administrador):

[System.Environment]::SetEnvironmentVariable("Path", $env:Path + ";C:\desarrollo\flutter\bin", "Machine")

Verificar la instalación:

flutter --version

La salida debe mostrar la versión de Flutter y Dart instaladas.

3.2 Instalar Android Studio

  1. Descargar Android Studio desde: https://developer.android.com/studio
  2. Ejecutar el instalador y seguir los pasos del asistente.
  3. Durante la instalación, asegurarse de marcar las siguientes opciones:
    • Android SDK
    • Android SDK Platform
    • Android Virtual Device (AVD)
  4. Al finalizar, abrir Android Studio y completar la configuración inicial del SDK.

3.3 Configurar Flutter con Android Studio

En la terminal, ejecutar el diagnóstico de Flutter para identificar dependencias faltantes:

flutter doctor

Los elementos más comunes que requieren acción:

Aceptar las licencias de Android:

flutter doctor --android-licenses

Instalar el plugin de Flutter en Android Studio desde el menú Plugins.

3.4 Instalar Git

  1. Descargar Git desde: https://git-scm.com/downloads
  2. Ejecutar el instalador con la configuración por defecto.
  3. Verificar la instalación:
git --version

05

Clonación del proyecto

Abrir una terminal en el directorio donde se desea guardar el proyecto y ejecutar:

git clone [URL_DEL_REPOSITORIO]
cd presupuesto_app
Nota: Reemplazar [URL_DEL_REPOSITORIO] con la URL real del repositorio Git del proyecto. Este dato debe ser proporcionado por el responsable del proyecto.

Verificar que la clonación fue exitosa listando el contenido del directorio:

# En Windows (PowerShell)
dir

# En macOS / Linux
ls -la

Se deben visualizar archivos como pubspec.yaml, lib/, android/, ios/, entre otros.

06

Instalación del proyecto

6.1 Instalar dependencias de Flutter

Desde la raíz del proyecto (presupuesto_app/), ejecutar:

flutter pub get

Este comando descarga e instala todos los paquetes declarados en pubspec.yaml. La salida esperada al finalizar es:

Got dependencies!

Paquetes principales del proyecto:

6.2 Generar código automático con build_runner

El proyecto utiliza hive_generator para generar adaptadores de tipo (archivos .g.dart) que Hive necesita para serializar y deserializar los modelos. Este paso es obligatorio antes de compilar. 

dart run build_runner build --delete-conflicting-outputs

La salida esperada al finalizar es similar a:

07

Base de datos Hive

La aplicación no requiere instalación ni configuración de una base de datos externa. Utiliza Hive, una base de datos NoSQL ligera y embebida que se almacena directamente en el sistema de archivos del dispositivo o emulador.

7.1 Inicialización automática

La base de datos se inicializa automáticamente en el primer arranque a través de lib/main.dart:

  1. Inicializa Hive con Hive.initFlutter().
  2. Registra los 10 adaptadores de tipo (typeIds del 0 al 9).
  3. Abre las cajas (colecciones) de datos:
    • proyectos — almacena objetos de tipo Proyecto
    • presupuestos — almacena objetos de tipo Presupuesto
    • materiales_catalogo — gestionada por MaterialesService
  4. Carga el catálogo inicial de 18 materiales si la caja está vacía (seed data automático).
No se requiere ejecutar ningún comando SQL ni migración manual.

7.2 Ubicación física de los datos

Hive guarda los datos en archivos .hive dentro del directorio de datos de la aplicación:

  • Android (emulador): /data/data/com.example.presupuesto_app/app_flutter/
  • Windows (escritorio): %APPDATA%\presupuesto_app\

No es necesario conocer esta ruta para el uso normal del sistema.

Antes de ejecutar, confirmar que hay un dispositivo o emulador disponible:

flutter devices

La salida mostrará los dispositivos conectados. Ejemplo:

Desde la raíz del proyecto, ejecutar:

flutter run

09

Verificación de la instalación

Una vez que la aplicación inicia correctamente, realizar las siguientes comprobaciones para confirmar que la instalación funciona bien.

Pruebas funcionales

Pantalla de splash: Al iniciar, debe aparecer una pantalla de bienvenida (SplashScreen) con el nombre de la aplicación. Confirma que Flutter carga correctamente.

Pantalla principal (Home): Debe aparecer la pantalla principal con una barra de navegación inferior con tres secciones: Proyectos, Presupuestos y Materiales. Si la barra es visible y responde al toque, la interfaz es correcta.

Catálogo de materiales (Hive): En la sección Materiales debe aparecer una lista con al menos 18 materiales precargados. Si la lista tiene contenido, Hive se inicializó correctamente.

Proyecto de prueba: Ir a Proyectos, agregar un nuevo proyecto con el nombre "Proyecto de verificación" y confirmar que persiste al regresar a la pantalla.

Checklist de verificación

Completado: 0 / 7
  • flutter doctor sin errores críticos — todos los checks en verde o amarillo aceptable
  • flutter pub get exitoso — salida: Got dependencies!
  • build_runner sin errores — salida: Succeeded after Xs
  • Splash screen visible al iniciar — pantalla de bienvenida cargada correctamente
  • Barra de navegación inferior visible — 3 secciones funcionales (Proyectos, Presupuestos, Materiales)
  • Catálogo de materiales con datos precargados — 18 o más materiales visibles
  • Proyecto de prueba guardado correctamente — persiste en la lista al navegar
Marca cada paso completado para registrar tu progreso.
10

Problemas comunes y soluciones

Causa: Flutter SDK no está agregado al PATH del sistema.

  1. Verificar que la ruta flutter\bin existe en el directorio donde se extrajo el SDK.
  2. Agregar esa ruta al PATH del sistema.
  3. Cerrar y volver a abrir la terminal.
  4. Ejecutar flutter --version para confirmar.

Causa: Android Studio no terminó de instalar el Android SDK, o la ruta no está configurada.

  1. Abrir Android Studio.
  2. Ir a Settings > Appearance & Behavior > System Settings > Android SDK.
  3. Verificar que hay al menos una plataforma SDK instalada (API 21 o superior).
  4. Copiar la ruta del SDK Location.
  5. En la terminal, configurar la variable de entorno:
flutter config --android-sdk "RUTA_DEL_SDK"

Mensaje: Some Android licenses not accepted.

Ejecutar y responder y en cada pregunta hasta que todas las licencias queden aceptadas:

flutter doctor --android-licenses

Causa: Sin acceso a internet no se pueden descargar los paquetes de pub.dev.

  1. Verificar la conexión a internet.
  2. Si hay proxy corporativo, configurarlo:
set HTTPS_PROXY=http://usuario:password@proxy:puerto
flutter pub get

Mensaje: Existing conflict between... o archivos .g.dart desactualizados.

Usar la bandera --delete-conflicting-outputs para que el generador sobreescriba los archivos en conflicto:

dart run build_runner build --delete-conflicting-outputs

Mensaje: No connected devices. Attach your device or start an emulator.

  1. Emulador Android: Abrir Android Studio > Device Manager > iniciar un AVD.
  2. Dispositivo físico Android: Activar Opciones de desarrollador y Depuración USB, luego conectarlo por USB.
  3. Windows escritorio: Habilitar la plataforma Windows:
flutter config --enable-windows-desktop

Causa: El plugin image_picker requiere permisos de cámara/galería en Android que no están configurados.

Verificar que android/app/src/main/AndroidManifest.xml contiene:

<uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE"/>
<uses-permission android:name="android.permission.CAMERA"/>

Si no están presentes, agregar esas líneas antes de la etiqueta <application>.

Causa: Se intenta acceder a una caja Hive antes de que sea abierta, o los adaptadores no están registrados.

  1. Asegurarse de que el paso build_runner se ejecutó correctamente y los archivos .g.dart existen.
  2. No modificar el orden de inicialización en lib/main.dart.
  3. Hacer un flutter clean y volver a ejecutar flutter pub get y build_runner.

Mensaje: Port XXXX is already in use.

Especificar un puerto diferente:

flutter run -d chrome --web-port=8090

Mensaje: 'background' is deprecated and shouldn't be used.

Causa: El proyecto usa una propiedad de ColorScheme que fue deprecada en versiones recientes de Flutter.

Impacto: Es solo una advertencia, no impide la compilación ni la ejecución. No requiere acción inmediata.
11

Despliegue

La aplicación es una app móvil/escritorio nativa y no tiene componente web desplegable en la nube. Las opciones de distribución son:

Android — APK directo

Genera un APK de release para instalar directamente en dispositivos sin pasar por Google Play.

flutter build apk --release

Archivo generado en:

build/app/outputs/flutter-apk/app-release.apk

Copiar al dispositivo e instalar con permisos de "Fuentes desconocidas" activados.

Windows — ejecutable

Genera un ejecutable nativo para Windows de 64 bits.

flutter build windows --release

Ejecutable generado en:

build/windows/x64/runner/Release/presupuesto_app.exe

Distribuir copiando toda la carpeta Release/ al equipo de destino.

Google Play / App Store

No especificado en el proyecto. Requiere:

  • Cuentas de desarrollador en Google Play Console y Apple Developer Program.
  • Configuración de firma de aplicaciones.
  • Proceso de revisión de cada plataforma.
12

Desinstalación

12.1 Eliminar dependencias descargadas

flutter clean

Este comando elimina la carpeta build/ y los archivos de caché locales. Los paquetes de pub.dev cacheados se pueden eliminar borrando:

  • Windows: %LOCALAPPDATA%\Pub\Cache
  • macOS/Linux: ~/.pub-cache

12.2 Eliminar archivos generados por build_runner

dart run build_runner clean

O borrar manualmente todos los archivos con extensión .g.dart del directorio lib/.

12.3 Eliminar los datos de la aplicación (base de datos Hive)

  • Emulador Android: Device Manager > Wipe Data en el AVD, o desde el dispositivo: Ajustes > Aplicaciones > Presupuesto App > Almacenamiento > Borrar datos.
  • Windows: Eliminar los archivos .hive y .lock en %APPDATA%\presupuesto_app\.

12.4 Eliminar el proyecto del equipo de desarrollo

# Windows (PowerShell)
Remove-Item -Recurse -Force presupuesto_app

# macOS / Linux
rm -rf presupuesto_app

12.5 Desinstalar Flutter SDK (opcional)

  1. Eliminar la carpeta donde se extrajo el SDK (ejemplo: C:\desarrollo\flutter).
  2. Remover la entrada de flutter\bin del PATH del sistema.
  3. Desinstalar Android Studio desde el panel de aplicaciones del SO si ya no se necesita.
13

Conclusión

El proceso de instalación de Presupuesto App consta de los siguientes pasos principales:

Instalar Flutter SDK y configurar el PATH

Descargar el SDK, extraer en ruta sin espacios y agregar flutter\bin al PATH del sistema.

Instalar Android Studio con Android SDK y aceptar licencias

Usar flutter doctor --android-licenses para aceptar todas las licencias pendientes.

Instalar Git y clonar el repositorio

Clonar con git clone [URL_DEL_REPOSITORIO] y acceder al directorio del proyecto.

Ejecutar flutter pub get

Descargar e instalar todas las dependencias declaradas en pubspec.yaml.

Ejecutar dart run build_runner build

Generar los adaptadores de Hive (.g.dart) necesarios para la base de datos local.

Iniciar un emulador o conectar un dispositivo físico

Verificar disponibilidad con flutter devices antes de ejecutar.

Ejecutar flutter run

Iniciar la aplicación. Hive se inicializa automáticamente en el primer arranque.

No se requiere configurar base de datos externa, servidores ni variables de entorno. La aplicación es completamente autónoma y funciona de forma offline desde el primer arranque.
Ante cualquier problema durante la instalación, consultar la sección 10 de este manual que cubre los errores más frecuentes con sus soluciones paso a paso.
